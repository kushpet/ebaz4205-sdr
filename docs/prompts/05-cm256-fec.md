# Step 5 — Port cm256 FEC from `f4exb/cm256cc`

You're picking up `ebaz4205-sdr` (see `CLAUDE.md`). The current
`firmware/src/protocol/cm256.c` is a no-FEC pass-through stub; recovery
only works when `nb_fec_blocks == 0`. Real FEC is needed for lossy
networks.

## Read first

1. `CLAUDE.md` § "Known caveats" (cm256 stub)
2. `firmware/src/protocol/cm256.{c,h}` — current pass-through
3. `firmware/src/protocol/sdrangel_frame.{c,h}` — frame structure with
   FEC slots
4. Upstream: <https://github.com/f4exb/cm256cc> (`cm256.h`, `cm256.cpp`,
   plus `gf256.h`/`gf256.cpp` it depends on)

## What to do

1. Drop the upstream sources into `firmware/third_party/cm256/`. Keep
   the upstream license headers intact.
2. Port C++ → C: rename `.cpp` → `.c`, replace classes with structs +
   functions, drop SIMD intrinsics (Cortex-A9 NEON can be re-enabled
   later — for v1 keep scalar). Use the existing project's coding
   conventions (no dynamic allocation in hot path; static buffers).
3. Replace the body of `firmware/src/protocol/cm256.c` to call into
   the ported code. Keep the existing function signatures from
   `cm256.h` so nothing else needs to change.
4. Update `firmware/src/protocol/sdrangel_frame.c` to advertise
   `nb_fec_blocks > 0` in the meta block (e.g. `nb_original=128,
   nb_fec=16`).
5. Add the new sources to the build — Vitis's auto-discovery picks up
   anything under `firmware/src/`, so place the ported files there
   instead of `third_party/`, OR add an `add_files` entry. The
   simplest approach: just put `cm256_port.c` and `gf256_port.c`
   under `firmware/src/protocol/`.

## Build & verify

Firmware-only — see `docs/prompts/01-ddc-tlast.md` § "Build chain"
step 2 + 3.

Hardware check: artificially drop UDP packets (`tc qdisc add ... loss
5%`) on the host and confirm SDRAngel still decodes correctly.

When done: substantial commit (multi-file). Update `CLAUDE.md` to
remove the cm256 caveat, mark row 5 ✅ in `docs/NEXT_STEPS.md`.

This is the largest task on the list — budget multi-hour. Sonnet or
Opus recommended.
