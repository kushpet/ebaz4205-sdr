# EBAZ4205 SDR — Обновлённый план v2 (с DDC/DUC)

## Почему прямой проброс ADC→Ethernet невозможен

Сырой поток с ADC: 60 МГц × 2 байта = **120 МБ/с = 960 Мбит/с**.
Ethernet 100BASE-TX с учётом заголовков и эффективностью 75%: **~9.4 МБ/с**.

Превышение в **13 раз**. Решение — DDC/DUC в FPGA.

---

## Теория: DDC (Digital Down Converter) и DUC (Digital Up Converter)

### RX путь: DDC

```
ADC 12 бит @ 60 МГц
  │
  ▼
[NCO] ──sin/cos──▶ [Mixer] ──▶ I(t), Q(t) @ 60 МГц, 13 бит
  │                                │
  │ частота fc от ARM               ▼
  │                         [CIC Decimator, N=5, R=30..240]
  │                          нет множителей, только сумматоры
  │                          внутренняя разрядность: 12 + N·⌈log₂R⌉ бит
  │                                │
  │                                ▼
  │                         [CIC Компенсирующий FIR]
  │                          halfband, децимация ×2
  │                          исправляет droop CIC в полосе пропускания
  │                                │
  │                                ▼
  │                         I/Q @ fs/D МГц, 16+16 бит
  │                                │
  │                                ▼
  │                         AXI-Stream → DMA → DDR → ARM
  │
  └── NCO frequency word = fc × 2³²/ 60МГц (AXI-Lite регистр)
```

### TX путь: DUC

```
ARM → DDR → DMA → AXI-Stream
  │
  ▼
I/Q @ fs/D МГц, 16+16 бит
  │
  ▼
[Interpolation FIR] halfband, интерполяция ×2
  │
  ▼
[CIC Interpolator, N=5, R=15..120]
  │
  ▼
[Mixer] с [NCO] на частоте fc
  │
  ▼
Real 14 бит @ 60 МГц → DAC
```

### NCO (Numerically Controlled Oscillator)

- **Фазовый аккумулятор** 32 бита: `phase[n] = phase[n-1] + Δφ`
- Приращение фазы: `Δφ = round(fc × 2³² / fs)`
- Таблица синуса/косинуса: 1024 записей × 18 бит → 1 BRAM
- Разрешение по частоте: `fs / 2³² = 60 МГц / 4294967296 ≈ 0.014 Гц`

### CIC фильтр (Cascaded Integrator-Comb)

Структура (N секций, децимация R):
- **N интеграторов** на тактовой fs (рекурсивные сумматоры)
- **Субдискретизатор** ÷R
- **N comb-секций** на тактовой fs/R (разностные фильтры)

**Нет ни одного множителя** — только сложение. Идеально для FPGA.

Рост разрядности: `B_вых = B_вх + N × ⌈log₂ R⌉`

| N | R | Внутренняя разрядность (от 12 бит) |
|---|---|---|
| 5 | 30  | 12 + 5×5 = **37 бит** |
| 5 | 60  | 12 + 5×6 = **42 бит** |
| 5 | 120 | 12 + 5×7 = **47 бит** |

После CIC — усечение до 16 бит (с правильным округлением).

### Компенсирующий FIR (halfband)

CIC имеет droop (завал АЧХ) в полосе пропускания. Компенсирующий FIR:
- Halfband структура: каждый второй коэффициент = 0, вдвое меньше умножений
- Децимация ×2 (дополнительная)
- ~63 коэффициента, симметричные → 32 умножения → 10-11 DSP48

### Итог: Ethernet bandwidth после DDC

| Децимация D | Выходная частота | Ethernet | Макс. полоса сигнала |
|---|---|---|---|
| **30** | **2.000 МГц** | **64 Мбит/с** | **1.0 МГц** |
| 60  | 1.000 МГц | 32 Мбит/с | 0.5 МГц |
| 120 | 0.500 МГц | 16 Мбит/с | 0.25 МГц |
| 240 | 0.250 МГц | 8 Мбит/с  | 0.125 МГц |

**Рекомендация**: D = 30..120 программно (AXI-Lite регистр из ARM),
максимальная рабочая полоса **1 МГц** при D=30. SDRAngel поддерживает
динамическую смену decimation через REST API.

## Ресурсы Zynq7010 (xc7z010clg400)

**Доступно**: 80 DSP48E1, 17 600 LUT, 35 200 FF, 60 BRAM18.

| Блок | DSP48 | BRAM18 | LUT |
|---|---|---|---|
| NCO + Mixer (DDC) | 1 | 1 | ~50 |
| CIC N=5 (DDC) | 0 | 0 | ~500 |
| Компенс. FIR (DDC) | 10–11 | 5 | ~200 |
| **DDC итого** | **~12** | **6** | **~750** |
| DUC (симметрично) | ~12 | 6 | ~750 |
| AXI DMA + Interconnect | ~4 | 4 | ~1500 |
| PS7 wrapper | 0 | 0 | ~200 |
| **Всего** | **~28 (35%)** | **16 (27%)** | **~3200 (18%)** |

Ресурсов достаточно с большим запасом.

#### 1.6 RTL: NCO + Комплексный Mixer (общий для DDC и DUC)
**Уровень**: ПР

Генерирует: `hardware/rtl/nco.v`, `hardware/rtl/complex_mixer.v`, `hardware/sim/tb_nco.v`

**nco.v**:
- 32-битный фазовый аккумулятор
- Таблица sin/cos: 1024 × 18 бит (BRAM или distributed RAM)
- Входной AXI-Lite регистр: `freq_word[31:0]` — приращение фазы
- Выход: `sin_out[17:0]`, `cos_out[17:0]` @ 60 МГц

**complex_mixer.v**:
- Вход: `data_in[11:0]` (от ADC), `sin[17:0]`, `cos[17:0]`
- Выход: `I_out[12:0]`, `Q_out[12:0]` — два DSP48 умножения
- Для DUC: обратная операция (I×cos - Q×sin)

**Передать в запросе**: результаты пп. 1.3, 1.4

---

#### 1.7 RTL: CIC Decimator / Interpolator
**Уровень**: ПР

Генерирует: `hardware/rtl/cic_decimator.v`, `hardware/rtl/cic_interpolator.v`, `hardware/sim/tb_cic.v`

Параметризованный модуль (параметры N, R через generic):
- **cic_decimator.v**: N интеграторов @ fs, субдискретизатор ÷R, N comb @ fs/R
  - Внутренняя разрядность: `B_in + N*log2(R_max)` бит (параметр)
  - Выход усекается до 16 бит с правильным округлением
  - Программируемый R через AXI-Lite (принимаемые значения: 15, 30, 60, 120)
- **cic_interpolator.v**: зеркальная структура для DUC

**Передать в запросе**: результат п. 1.6 (разрядность на входе CIC)

---

#### 1.8 RTL: Компенсирующий FIR (halfband)
**Уровень**: ПР

Генерирует: `hardware/rtl/hb_fir_decimator.v`, `hardware/rtl/hb_fir_interpolator.v`

Halfband FIR фильтр:
- 63 коэффициента (32 ненулевых), симметричные
- Децимация/интерполяция ×2
- Реализация на DSP48 (транспонированная форма)
- Коэффициенты: compensate CIC droop для R ≤ 120 в полосе до fs/2D

Опционально: коэффициенты загружаются из DDR через AXI-Lite для
поддержки разных R (Parks-McClellan на ARM, как в Maia SDR).

**Передать в запросе**: результат п. 1.7 (выходная разрядность CIC, 16 бит)

---

#### 1.9 RTL: DDC топ-модуль (RX)
**Уровень**: ПР

Генерирует: `hardware/rtl/ddc_top.v`

Соединяет: adc_if → complex_mixer (с NCO) → cic_decimator → hb_fir_decimator → AXI-Stream I/Q

Регистровая карта AXI-Lite (единый блок управления):
- `0x00`: `nco_freq_word[31:0]` — центральная частота
- `0x04`: `decimation_rate[7:0]` — R (15/30/60/120)
- `0x08`: `status[31:0]` — overflow, link, lock

**Передать в запросе**: результаты пп. 1.4, 1.6, 1.7, 1.8

#### 2.5 Управление NCO из ARM (новый пункт)
**Уровень**: О

Генерирует: `firmware/src/platform/ddc_ctrl.c/h`

Функции:
- `ddc_set_frequency(uint32_t fc_hz)` → вычисляет freq_word, пишет в AXI-Lite

- `ddc_get_status()` → читает статус overflow/lock

Вычисление freq_word: `freq_word = (uint64_t)fc_hz * (1ULL << 32) / 60000000ULL`
